When I attach a data frame R says that some variables are "masked."

R uses the word "mask" to represent something that is multiply defined. For example, when you load the NCStats package R will tell you that the function "Summary" is masked. This occurs because the NCStats package contains a function called "Summary" AND the function "Summary" appears in the base R. In this instance, the masking is not a problem because in statistics classes at Northland you will only need the "Summary" function in the NCStats package.

Masking can become a problem when a variable (rather than a function) is masked. This is a problem because R may become confused about which variable to use (actually, it is more likely that you will get confused). This issue usually occurs when you have two data frames with the same variable names attached at the same time. For example, consider this simple example ...

> str(d1)
'data.frame':   3 obs. of  4 variables:
 $ classes : int  3 4 1
 $ name    : Factor w/ 3 levels "Derek Ogle","Jefry Johnson",..: 1 3 2
 $ years   : int  11 19 3
 $ hometown: Factor w/ 3 levels "Ashland","Duluth",..: 3 2 1
> attach(d1) 
> str(d2)
'data.frame':   3 obs. of  3 variables:
 $ classes: int  3 4 1
 $ name   : Factor w/ 3 levels "Derek Ogle","Jefry Johnson",..: 1 3 2
 $ years  : int  11 19 3
> attach(d2)

        The following object(s) are masked from d1 :

         classes name years

If this type of masking occurs it is best to detach the data frame that you do not want to use (see this FAQ). If you cannot remember which data frames you have attached then see this FAQ.
