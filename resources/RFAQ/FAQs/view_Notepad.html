When I view my text file in Notepad it looks like data is missing.

Many .txt data files, when viewed in Notepad will look like they either have more or less data than is actually needed or should be in the data file. For example, the data file below has two columns of data, but the columns do not line up neatly so it looks like some data is missing.

First.Name      Last.Name
Derek   Ogle
Young   Kim
Jefry   Johnson

The "problem" is that the header for the first column is longer than any of the data in the first column so the tab that follows it causes the header for the second column to start on a different tab stop than the data for the second column. This is not a problem because R reads in data by separating columns by finding "white space". Thus, the two column headers are separated by one white space and each of the data rows is separated by just one white space. Thus, these data will be reading into R correctly.

> d1 <- read.table("Ex1_spaces1.txt",header=T)
> str(d1)
'data.frame':   3 obs. of  2 variables:
 $ First.Name: Factor w/ 3 levels "Derek","Jefry",..: 1 3 2
 $ Last.Name : Factor w/ 3 levels "Johnson","Kim",..: 3 2 1
> d1
  First.Name Last.Name
1      Derek      Ogle
2      Young       Kim
3      Jefry   Johnson
